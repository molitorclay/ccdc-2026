Snort & Fail2Ban:
Configure snort to listen on server range
Make a list of rules 
similar to: https://github.com/mandiant/red_team_tool_countermeasures/blob/master/all-snort.rules
Configure Fail2Ban to filter snort
Create /etc/fail2ban/filter.d/snort.conf:
failregex = ^.*?\s+\[Drop\s+Rules\s+ID\s+.*?\]\s+.*?\s+->\s+.*?\s+<HOST>
                  ^.*?\s+\[\d+:\d+:\d+\]\s+.*?\s+->\s+.*?\s+<HOST>
ignoreregex =
Note: The failregex must match your specific alert_fast log format.
Create Fail2Ban jail
create/edit /etc/fail2ban/jail.local
[snort-linear] enabled = true port = all filter = snort logpath = /var/log/snort/alert_fast maxretry = 1 bantime = 3600 action = iptables-allports[name=Snort]
Sudo systemctl restart fail2ban
